## 1. 제품 개요

이메일/비밀번호 인증 기반의 Todo 웹앱. 사용자별로 할 일을 생성, 조회, 수정, 삭제할 수 있는 MVP.

---

## 2. 목표

- 빠르게 할 일을 기록하고 관리
- 심플한 UX로 최소한의 마찰
- 새로고침 후에도 데이터 유지
- 사용자별 데이터 격리

---

## 3. 사용자 스토리

| ID | 스토리 |
|----|--------|
| US-01 | 사용자는 이메일과 비밀번호로 회원가입할 수 있다 |
| US-02 | 사용자는 이메일과 비밀번호로 로그인할 수 있다 |
| US-03 | 사용자는 로그아웃할 수 있다 |
| US-04 | 사용자는 새 Todo를 추가할 수 있다 |
| US-05 | 사용자는 Todo 목록을 볼 수 있다 |
| US-06 | 사용자는 Todo를 완료 처리할 수 있다 |
| US-07 | 사용자는 Todo 내용을 수정할 수 있다 |
| US-08 | 사용자는 Todo를 삭제할 수 있다 |

---

## 4. 기능 요구사항

### 4.1 회원가입 (US-01)
- 이메일 형식 검증
- 비밀번호 최소 6자 이상
- 이미 사용 중인 이메일은 가입 불가
- 가입 즉시 로그인 상태로 전환

### 4.2 로그인 (US-02)
- 이메일 + 비밀번호 입력
- 불일치 시 오류 메시지 표시
- 로그인 성공 시 세션 유지 (새로고침 후에도 유지)
- 로그인/회원가입 폼을 한 화면에서 전환 가능

### 4.3 로그아웃 (US-03)
- 헤더의 로그아웃 버튼으로 즉시 세션 종료
- 로그아웃 후 로그인 화면으로 이동

### 4.4 Todo 생성 (US-04)
- 텍스트 입력 후 Enter 또는 추가 버튼 클릭으로 생성
- 빈 문자열 또는 공백만 있는 경우 추가 불가
- 추가 후 입력 필드 자동 초기화

### 4.5 Todo 목록 조회 (US-05)
- 로그인한 사용자의 Todo만 표시 (사용자별 데이터 격리)
- 전체 목록을 최신 추가순으로 표시
- 완료/미완료 상태를 시각적으로 구분 (색상, 취소선)
- 미완료 항목 수 헤더에 표시

### 4.6 완료 처리 (US-06)
- 각 항목에 체크박스로 완료 토글
- 완료된 항목은 취소선 및 흐린 색상으로 표시
- 완료 상태는 즉시 저장
- 완료 항목 일괄 삭제 버튼 (완료 항목 존재 시만 표시)

### 4.7 Todo 수정 (US-07)
- 편집 아이콘 클릭으로 인라인 편집 모드 진입
- 편집 중 Enter → 저장, ESC → 취소
- 빈 문자열로 저장 불가

### 4.8 Todo 삭제 (US-08)
- 각 항목에 삭제 버튼 제공
- 클릭 시 즉시 목록에서 제거 (확인 다이얼로그 없음)

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| 데이터 지속성 | 브라우저 localStorage에 저장, 새로고침 후에도 유지 |
| 데이터 격리 | Todo는 사용자 ID 단위로 분리 저장 |
| 반응형 | 모바일(320px~) 및 데스크탑 모두 지원 |
| 성능 | 모든 인터랙션은 즉각 반응 (로딩 없음) |
| 접근성 | 키보드만으로 전체 조작 가능 |

## 6. 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프레임워크 | React 18 |
| 번들러 | Vite 5 |
| 스타일 | CSS3 (Flexbox, CSS Variables) |
| 저장소 | Browser localStorage |

## 7. 범위 외 (Out of Scope for MVP)

- 서버 또는 외부 DB 연동 (현재 localStorage 기반)
- OAuth / 소셜 로그인
- 비밀번호 찾기 / 재설정
- 이메일 인증
- 카테고리 / 태그 / 라벨
- 우선순위 설정
- 마감일 / 알림
- 드래그 앤 드롭 정렬
- 다중 목록 (프로젝트)

---

## 8. 보안 고려사항

> **MVP 한계**: 현재 비밀번호는 `btoa()` 인코딩 후 localStorage에 저장됩니다. 실제 서비스 배포 시에는 반드시 백엔드 인증(JWT, OAuth 등)과 서버사이드 해싱으로 교체해야 합니다.

---

## 9. 성공 지표

- 회원가입 → 로그인 → Todo CRUD 전체 흐름이 오류 없이 작동
- 새로고침 후 로그인 세션 및 Todo 데이터 복원 확인
- 다른 계정 로그인 시 Todo 데이터가 분리되어 표시
- 모바일 뷰포트(375px)에서 UI 깨짐 없음
